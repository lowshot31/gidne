## 🚀 배포 완수 내역

2. **[아키텍처]** Data Pump Bot 도입 (야후 금융/매크로 중앙 집계 및 캐싱 인프라)
3. **[기능]** 매크로 포커스 (M2, VIX, 환율 등 필터링 기반 특화 검색 모드 도입)

5. **[방어]** 야후 429 에러 방어(0.5초 Throttling, 15초 Circuit Breaker)
6. **[프라이버시]** AI 크롤링 봇 원천 차단 (`robots.txt` 하드코딩 도입)

## 📚 깨달은 점 / 교훈
1. **Cloudflare Edge(Workers) 제약**: 배포 환경에서 Node.js 패키지 직접 접근이나 크롤링을 시도하면 타임아웃 및 IP 블로킹을 맞을 확률이 높다. 
2. **탈중앙화 캐시 전략 승리**: 무거운 수집은 로컬/별도 봇(`data-pump.ts`)이 짊어지고, 배포 서버(CF Pages)는 Redis만 빠르게 읽고 빠지는 구조가 퍼포먼스면에서 압도적이다.

## ⚠️ 기술 빚 (다음 주 바로 잡을 것)
- **로컬 의존성 제거**: 현재 `data-pump.ts`를 개인 PC에서 꺼버리면 배포 서버의 실시간 데이터 갱신이 멈춤 (다음 주 즉시 클라우드에 봇 배포 요망).
