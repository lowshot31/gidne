## 🚀 배포한 핵심 아키텍처 (Architecture Updates)
─────────────────────
이번 스프린트와 리뷰 과정에서 결정된 주요 아키텍처 설계 및 도입 사항은 다음과 같습니다:

### 1. Flexbox 기반 다이나믹 벤토(Bento) 그리드
- **문제점:** 반응형 그리드(`react-grid-layout`)에서 위젯 크기를 조절할 때 컨텐츠가 잘리거나 스크롤이 어긋나는 현상.
- **해결책 (설계 원칙):** 모든 메인 대시보드의 위젯(`DashboardClient` 하위 컴포넌트)은 내부적으로 `display: flex; flex-direction: column`과 `flex: 1`을 반드시 가지도록 강제. 
- **결과:** 어떠한 해상도나 커스텀 그리드 설정에서도 UI가 유동적(Fluid)으로 늘어나고 줄어들며 완벽한 정보 레이아웃 밀도를 유지.

### 2. 무중단 캐시 실패 대처 (Graceful Error Handling)
- **문제점:** 백그라운드 스크립트(`data-pump.ts`)가 꺼져 있거나 Redis가 웜업(Warm-up) 중일 때, 대시보드 전체가 빨간 에러창으로 덮이며 작동 불능 상태에 빠짐.
- **해결책:** `DashboardClient`의 메인 렌더링 블로킹을 해제. 데이터를 받지 못한 경우 전체를 가리는 에러창이 아니라, **"시황 데이터 집계 중" 벤토 스타일 오버레이**를 띄우도록 우아하게(Graceful) 폴백(Fallback) 처리.
- **결과:** 워치리스트나 개별 뉴스 등 시장 데이터 전역 캐시와 무관한 위젯은 영향을 받지 않고 계속 사용 가능 (가용성 극대화).

### 3. 클라이언트 사이드 스마트 데이터 매니징
- **Watchlist 드래그 앤 드롭:** 무거운 서드파티 라이브러리 없이 순수 HTML5 Drag & Drop API(`onDragStart`, `dragOver`)와 React 상태(`isDragging`)를 결합. 배열 요소의 교환(Splicing) 결과를 신속히 LocalStorage에 동기화.
- **뉴스 파서 통일성:** API가 밀리초(Date.now()), 초 단위 UNIX 시간, ISO 날짜 문자열(`YYYY-MM-DD...`)을 혼용하여 반환할 때 발생하는 `NaN일 전` 렌더링 버그를 차단하는 지능형 `timeAgo` 파서 전역 적용.

## 📚 배운 점 & ⚠️ 기술 부채 유의점
─────────────────────
1. **서버 렌더링 캐시(SSR/API)와 브라우저 시간의 동기화:** 
   브라우저 시계와 서버 시계의 오차가 있을 경우 시간이 음수(마이너스)로 파싱되어 에러를 뿜을 수 있다는 엣지 케이스 발견(해결 완료).
2. **야후 파이낸스의 Rate-Limiting 한계성 파악:** 
   Watchlist 개수를 30개로 캡핑하고, API에 500ms 딜레이를 주었음에도 불구하고 과도한 요청 시 429 에러가 심심찮게 일어남. 결국 클라이언트 직접 호출을 줄이고 `data-pump.ts` 백그라운드가 긁어와 Redis로 서빙하는 중간 매개체 아키텍처의 당위성을 재확인.
3. **UX 철학 (토스증권 및 마진킹 벤치마킹):** 
   정보를 숨기지 않고 가장 눈에 잘 띄는 곳(헤더나 최상단)에 압축 배치하는 것의 위력. EOD 브리핑에서 관심종목 Movers를 맨 위로 빼고, 개별 종목 차트에 1D/52W 위치(Dot 바)를 직접 노출하자 활용도가 급상승함.
