# EOD Alpha Briefing v2.0 기능 명세서 (Feature Specification)

본 문서는 새롭게 고도화/통합된 **EOD Alpha Briefing 위젯**과 **Watchlist(관심종목) 파이프라인**의 상세 아키텍처 및 세부 기능을 정의합니다.

---

## 1. 개요 (Overview)
금융 대시보드(Gidne)의 핵심 기능인 EOD Alpha Briefing에서 기존의 당일(1D) 데이터뿐만 아니라 주간(1W), 월간(1M) 모멘텀을 분석할 수 있도록 아키텍처를 확장했습니다. 또한, 사용자 개인의 관심종목(Watchlist) 데이터를 통합 분석하여 커스텀 수익률 대장주를 추출해주는 기능을 성공적으로 구현했습니다.

---

## 2. 주요 기능 스펙

### A. 동적 타임프레임 전환 (Dynamic Timeframe)
* **기능 설명**: EOD 브리핑 상단의 `1D`, `1W`, `1M` 탭을 클릭하여 전체 대시보드의 기준 기간을 동적으로 전환합니다.
* **적용 대상**: S&P 500 벤치마크 지수, 전체 섹터 등락률, 개별 종목, 관심 종목 등 EOD 화면 내의 모든 % 수치.
* **아키텍처 (On-Demand Caching)**:
  1. 프론트엔드가 변경된 타임프레임에 필요한 종목 `Ticker` 배열을 수집하여 백엔드에 요청합니다.
  2. 서버 엔드포인트(`/api/returns`)가 요청받은 종목에 대해 최대 2달 치의 과거 가격 차트를 분석합니다.
  3. 실시간으로 1주일 전, 1달 전 종가 대비 등락률(`change1W`, `change1M`)을 계산하여 반환합니다.
* **캐시 전략**: 계산된 수치는 Redis 키(`gidne_ret_v2_<ticker>`)에 **4시간 동안 보관(TTL)**되어 중복 연산을 방지하고 브라우저 전환 속도를 극대화합니다.

### B. 개인 워치리스트 Top Movers (Personalized Momentum)
* **기능 설명**: 시장 전체의 거시적 섹터 데이터 아래에, 사용자 본인이 담아둔 관심 종목 중 제일 크게 움직인 종목을 나열합니다.
* **출력 방식 (Accordion UI)**:
  * **기본 모드**: 수많은 관심 종목 중 당일 주도주(상위 3개)와 폭락주(하위 1개)만 선별하여 가장 눈에 잘 띄게 스마트 압축 노출합니다.
  * **확장 모드**: 우측 끝의 `더보기 ▼` 버튼 클릭 시, 사용자가 등록한 모든 워치리스트 종목의 수익률 순위가 부드럽게 펼쳐집니다.
* **상호작용 연동**: 1W나 1M 탭 클릭 시, 관심 종목의 대장주 순위도 주간/월간 수익률을 기준으로 완벽히 재배치됩니다.

### C. 워치리스트 캡핑 (API Rate Limit Defense)
* **기능 설명**: `Watchlist`에 사용자가 등록할 수 있는 관심종목의 최대 개수를 하드 코딩으로 **30개**로 제한합니다.
* **기획 의도**: 워치리스트가 무제한일 경우 사용자가 `1M` 등락률을 눌렀을 때 순간적으로 수백 개의 과거 차트를 호출하게 되어 야후 파이낸스 측에서 스크래핑 IP를 영구 차단(HTTP 429)할 위험이 높습니다. 
* **에러 핸들링**: 사용자가 31번째 종목을 추가하려 하면 빨간색 인라인 경고("워치리스트는 최대 30개까지만 추가 가능합니다.") 텍스트를 즉각 렌더링하고 로컬 업데이트를 방지합니다.

---

## 3. 기술적 세부 사항 (Technical Architecture)
* **API Endpoints**: 
  * `GET /api/returns?tickers=AAPL,TSLA...` (최대 100개 배치 단위 쿼리)
* **백엔드 쓰로틀링 (Throttling)**: 캐시 미달(Cache Miss)로 차트를 긁어와야 할 경우 비동기 루프 안에서 1 종목당 `200ms`의 의도적인 지연(Sleep)을 발생시켜 서버 밴(Ban) 확률을 최소화.
* **프론트 UI 렌더링 원칙 (Flexbox 통일)**: 모든 대시보드 위젯 컴포넌트는 `display: flex; flex-direction: column` 구조를 띄며 `flex: 1`을 통해 외부 반응형 그리드 래퍼 크기에 맞추어 유동적으로 변환되게 설계됨.
* **Fall-back 오버레이 UI**: 초기 데이터 펌프가 미완료되어 Redis가 비어있을 시 컴포넌트 렌더링이 블로킹되는 현상을 막기 위해, 에러 발생 시 대시보드 구조를 파괴하지 않고 부드러운 '시황 데이터 집계 중 (Loading Overlay)' 형태의 UI로 우아한 대처(Graceful Degradation)를 적용.
