# Gidne v2 - 고성능 금융 대시보드 터미널

Gidne v2는 일반적인 주식 트래커 수준을 넘어, 전문가 수준의 트레이딩 터미널급 기능과 속도를 제공하는 현대적인 금융 대시보드입니다. Redis 데이터 캐싱과 백그라운드 데이터 펌프(Data Pump)를 통해 병목 현상 없는 실시간 시황을 제공합니다.

## 🚀 주요 핵심 기능

### 1. 매일 쏟아지는 인사이트: Alpha EOD Briefing
매크로 경제와 섹터 간의 상관관계를 자동으로 분석해주는 장 마감(EOD) & 장중 브리핑 시스템입니다.
- **상대강도(RS) 분석:** S&P 500 및 NASDAQ 대비 1일/1주일/1개월 기준 섹터 및 개별 종목의 상대적 강세를 추적합니다.
- **원클릭 텍스트 복사:** 주도 섹터, 소외 섹터, 그리고 매크로 드라이버를 메신저에 바로 공유할 수 있도록 클립보드 복사 템플릿을 제공합니다.
- **내 관심종목 Movers:** 워치리스트에 등록된 종목 중 급등/급락 종목을 브리핑 최상단에서 우선 확인합니다.

### 2. 살아 숨쉬는 Watchlist (드래그 앤 드롭 지원)
- 개인정보 보호를 위한 로컬 스토리지 기반 개인화 워치리스트.
- **자유로운 정렬:** 마우스로 종목을 드래그 앤 드롭(Drag-and-Drop)하여 원하는 순서대로 자유롭게 배치할 수 있습니다.
- 야후 파이낸스 API 횟수 제한을 우회하기 위한 스마트 자동완성 캐싱 기술 탑재.

### 3. 직관적인 토스증권 스타일 차트 헤더
- 복잡한 정보를 1초 만에 파악할 수 있는 고밀도 시각화 헤더.
- 1일 및 52주 최저/최고점 범위 내에서 현재 주가의 위치를 빛나는 점(Dot) 슬라이더 바 형태로 시각화.
- 단순 가격뿐만 아니라 시가총액, 거래량 비율(평균 대비), 상장 시장 등을 컴팩트하게 제공.

### 4. 무중단 캐싱 파이프라인 (`data-pump.ts` & Redis)
- 잦은 API 호출로 인해 발생하는 429(Too Many Requests) 에러를 원천 차단.
- 백그라운드 워커가 24시간 내내 매크로, 섹터, 시장 폭(Breadth), 암호화폐 데이터를 미리 수집하고 규격화합니다.
- **우아한 에러 핸들링:** 로컬 서버를 막 켜서 Redis에 아직 데이터가 없을 경우, 보기 싫은 빨간색 화면이 아닌 Bento 스타일 디자인에 완전히 녹아드는 "시황 데이터 집계 중" 대기 화면을 보여줍니다.

### 5. 실시간 뉴스 & 스쿼크(Squawk) 통합 연결
- 야후 파이낸스의 심층 뉴스와 실시간 뉴스 피드 완벽 연동.
- **스마트 시간 파서 도입:** 백엔드에서 날짜를 밀리초, 초, 또는 ISO-string(`2024-04-07T...`) 등 어떤 포맷으로 던지더라도 `방금 전`, `NaN일 전 오류 수정` 등 프론트엔드가 완벽하게 해석하여 처리합니다.
- 느린 FT RSS를 과감히 폐기하고, 아웃바운드 링크를 통한 실시간 초단위 `saveticker.com` 터미널 즉시 연결 기능을 도입했습니다.

### 6. 다이나믹 벤토(Bento) UI 그리드
- `react-grid-layout`을 사용하여 사용자가 자유롭게 위젯을 드래그하고 사이즈를 조절할 수 있습니다.
- 완벽한 Flexbox 적용으로, 위젯 크기를 극단적으로 늘리거나 줄이더라도 스크롤과 레이아웃이 절대로 깨지지 않습니다.
- Space Grotesk 특수 폰트와 JetBrains Mono 등 고급 타이포그래피를 적용한 세련된 모던 다크모드.

## 🛠 사용된 기술 스택
- **프론트엔드 환경:** Astro.js, 핵심 위젯은 React Islands (`client:load`)
- **스타일링:** Vanilla CSS (Tailwind 배제, 완벽한 커스텀 통제 달성)
- **상태 & 데이터 스트림:** Redis 메모리 캐시 / Finnhub WebSockets
- **실행 환경:** NPM / 로컬 Node 데스크톱 환경

## ⚙️ 실행 방법 (Getting Started)

아래 명세된 대로 UI 서버와 데이터를 펌핑하는 백그라운드 스크립트를 같이 실행해야 정상 작동합니다.

\`\`\`bash
# 1. 패키지 설치
npm install

# 2. Redis 백그라운드 데이터 펌프 가동 (필수)
npx tsx --env-file=.env scripts/data-pump.ts

# 3. 대시보드 서버 실행 (새로운 터미널에서)
npm run dev
\`\`\`
